PAPI §5 sign-challenge — browser demo

Proves the desktop spike for papi#36. The browser talks only to the local papi sign-challenge-serve oracle's /login endpoint; the oracle runs the whole RFC-0001 §5 handshake (discovery → challenge → response) against the PAPI server server-side — so no PAPI-server CORS is needed — signing the challenge with the card (slot-9A) and returning the minted PiggySession. The card never leaves the oracle's host; the browser never sees the private key.
